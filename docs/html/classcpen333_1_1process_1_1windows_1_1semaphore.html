Detailed Description

Inter-process named semaphore primitive.

Used to limit access to a number of resources. Contains an integer whose value is never allowed to fall below zero. There are two main supported actions: wait(), which decrements the internal value, and notify() which increments the value. If the value of the semaphore is zero, then wait() will cause the thread to block until the value becomes greater than zero.

This implementation has no explicit maximum value

This semaphore has USAGE PERSISTENCE, meaning the mutex will continue to exist as long as at least one process/thread is holding a reference to it.
